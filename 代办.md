MVP：
- text/plain 文本同步
- 设备配对
- 本地安全存储
- 最近记录
- 状态显示
- 设置页
- 深色模式
- 基础日志与错误恢复

V1：
- 自动同步 / 手动发送模式
- 本地历史记录
- 黑名单应用 / 敏感内容过滤
- 多语言（中文优先，英文次之）
- 快捷键 / 快速粘贴
- Windows 托盘 / macOS 菜单栏（如适合）
- Android 通知快捷操作
- iOS 分享扩展 / 快捷指令（如可行）

V2：
- 图片同步
- 富文本同步
- 文件传输
- 云端恢复策略
- 多空间支持
- 条件同步（仅 Wi-Fi / 仅充电时 / 仅前台等）
- 支持webdev同步剪贴板和账户设置
- 支持全平台应用变为服务器
- 全平台应用支持所有的功能

### Windows
要求：
- 符合最新 Windows 现代桌面体验
- 设置页、状态页、设备列表页、同步记录页要结构清晰
- 支持系统深浅色、窗口行为、快捷键、通知
- 若适合，支持系统托盘或后台驻留能力

### Android
要求：
- 使用 Android 最新设计体系与原生交互习惯
- 支持深浅色、动态色彩、无障碍
- 考虑系统剪贴板访问限制、后台运行限制、通知权限、前台服务等问题
- 必须对不同 Android 版本差异进行设计

### iOS / iPadOS
要求：
- 遵循 Apple HIG 和原生交互模型
- 考虑剪贴板访问提示、前后台切换、系统隐私策略
- 如果系统限制导致无法实现真正后台监听，必须提供合规替代方案
- 可选支持分享扩展 / 快捷指令 / Widget 等增强能力，但不要影响 MVP

### macOS
要求：
- 遵循 macOS 桌面范式
- 可考虑菜单栏驻留、快捷键、窗口化设置、通知、历史记录
- 支持桌面用户高效工作流

### 通用要求
- 中文优先，保留英文国际化能力
- 深色 / 浅色主题
- 无障碍支持
- 键盘可操作
- 日志 / 状态 / 设置 / 设备管理必须清晰
- 不允许牺牲原生体验来换取“看起来开发更快”的方案
